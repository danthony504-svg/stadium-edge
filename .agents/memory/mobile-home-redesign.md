name: Mobile Home parlay-assistant redesign
description: How the stadium-mobile Home screen reproduces the web parlay-assistant layout under the never-fabricate rule, plus the Coach auto-send token pattern.
---

The mobile Home (`artifacts/stadium-mobile/app/(tabs)/index.tsx`) mirrors the web ParlayBuilder home layout: logo, search pill, tagline, "Build best parlay" button, three quick chips (Hot Picks / Easy Money / Lottery Ticket), Featured Players row, Live Now row, then the existing Upcoming odds list.

**Never-fabricate substitution (the key decision):** the web app's FEATURED PLAYERS show a hardcoded `form X/10` rating from a sample `PLAYERS` object — pure fabrication. The mobile App-Store build forbids that. Featured Players are instead built ONLY from real bookmaker props: fan out `getProps` over the soonest ~4 pickable games of the selected sport, take non-alt lines with real `overPrice`+`line`+`headshot`, dedupe by player, cap 8. The card's third line is the REAL prop (`o{line} {label} {price}`), never a form score.
**Why:** copying the web's fake form rating would violate the hard never-fabricate rule the whole mobile app exists to satisfy.
**How to apply:** any "featured/notable players" surface on mobile must derive its headline stat from a real feed (props/game-log), not a rating. Team abbr is resolved by matching `prop.playerTeamId` against the game's `homeTeamId`/`awayTeamId` → `homeAbbr`/`awayAbbr` (props feed has no abbr/position). Featured row is gated to `PROPS_SPORTS` only.

**Coach auto-send token pattern:** quick chips + Build button + Live "build from this game" navigate to `/coach` with `{prefill, send:"1", ts:Date.now()}`. coach.tsx auto-sends in an effect gated by the per-navigation `ts` token — NOT the prefill text — because tabs stay mounted and several actions share the same prompt text ("Build me the best parlay" is used by both Build button and Hot Picks); text-keyed guard suppresses later taps. Mark sent only after invoking send, and skip while `streaming` (effect re-runs when streaming flips false so the send isn't lost).

Featured-player tap and the search pill route to `/props` with `{q, sp}` params; props.tsx reads them via `useLocalSearchParams` + a sync `useEffect` to seed search query + sport selector.

**Featured ↔ Props source-alignment gotcha:** Featured Players MUST draw its game list from the SAME source + ordering as the Props tab (Odds API odds, soonest-first), not from ESPN `gamesQ` in arbitrary order. Otherwise a featured player can come from a game the Props tab never loaded (Props caps soonest 12; ESPN order is not chronological), and the deep-link search returns "No matching players". Featured uses the soonest-4 odds games (⊆ Props' soonest-12); ESPN games only supply team ids/abbrs (matched by `nickname(away)|nickname(home)`) for headshots + labels. Gate the featured query on `gamesQ.isFetched` so headshots/team-ids attach on the first pass.

**Soccer (and any yes/no prop) silently hid the whole section:** the extraction filter required BOTH a `headshot` AND a numeric `line`. Soccer has NEITHER — ESPN serves no soccer player headshots (0 of ~289 props had one) and the marquee market `player_goal_scorer_anytime` is yes/no with `line == null` — so every soccer prop was dropped and the section render gate (`featured.length > 0`) hid it entirely even though `PROPS_SPORTS` includes soccer. Fix: require ONLY a real `overPrice`; headshot is optional (the avatar already falls back to initials) and `line` may be null (`FeaturedPlayer.line: number | null`); the card renders `Over {line} {label}` when a line exists else just `{label} {price}` (e.g. "Anytime Goal +150"). Same relaxation also surfaces NFL Anytime TD. **Why:** a headshot/line gate is a quality preference, not a data requirement — gating on it fabricates nothing but makes whole real markets invisible. **How to apply:** when a Featured/notable surface is empty for one sport but not others, check the per-prop filter against that sport's real prop SHAPE (yes/no line-null, no headshot) before assuming the feed is empty — verify with a live `/api/sports/props` curl (count `withHeadshot`/`withLine`/`passFilter`).

**Avatar fallback tier — headshot → team logo → initials:** soccer (and any sport ESPN serves no player headshots for) left Featured Players as bare initials. The avatar now falls back through REAL imagery only: player headshot, then the player's team logo, then initials. Team logos already ride on the ESPN game feed (`g.homeLogo`/`g.awayLogo`) — thread them through `teamInfoMap` and resolve per player with the SAME `playerTeamId === home/awayTeamId` test used for `teamAbbr` (null when unmatched). `FeaturedAvatar` uses per-tier `onError` state so a dead headshot/logo URL drops to the next tier instead of leaving a blank. **Why:** team logo is real imagery (not fabrication) and far more recognizable than initials. **How to apply:** never add a synthetic/placeholder avatar; only chain real feed imagery.

## Premium redesign (hero parlay + AI rails)

The Home was later rebuilt into a premium layout while keeping every metric real-or-hidden. New sections (all derived from the SAME per-game featured `getProps` fan-out, flattened into `propEntries`, plus a `getLiveSteals` query):
- **Hero "Today's Best AI Parlay"** — top server-`ev` props filtered to `ev!=null && fairProb!=null`, ONE per distinct game (independence) + one per player, 2–3 legs. Combined odds = `parlayAmerican(real per-leg prices)`; "Fair win prob" = product of per-leg `fairProb` (labeled FAIR not "model" because `fairProb` is the de-vigged cross-book consensus, not a bespoke model); "Avg edge" = mean of per-leg `edge`. Hidden (plain Build button fallback) when `<2` legs qualify.
- **Hot Picks Today** rail — `recommendSide` candidates graded by REAL recent hit-rate via the shared `gradePropCands` (lib/propGrade.ts), sorted by grade then hit%, top 6. Grade query keyed on a stable `hotKey` string so it only refetches when the candidate set changes.
- **AI Track Record** card — auto-graded W/L of the app's OWN longshot steal picks (NOT user bets); win% = `wins/(wins+losses)` with **pushes EXCLUDED**; shown only when `record.graded > 0`.
- **Top Value Props** list — props with server `ev >= HOME_MIN_VALUE_EV` (1.5), deduped by player, top 5; EV is server-computed only, never recomputed client-side.
- Quick actions are now 5 horizontally-scrollable chips with per-chip `onPress` (Hot Picks/Easy Money→askCoach, Best Value→/props, Longshots→/steals, AI Parlays→/coach) — type changed from `{msg}` to `{onPress}`.

**Shared grading SoT:** `lib/propGrade.ts` (`GRADE_POOL`, `gradePropCands`, `recommendSide`, `GradeResult`) was extracted from props.tsx so Home + Props tab grade identically. Any grading change goes there once.

**Hook-order note:** the new `useQuery`s (`stealsQ`, `hotGradesQ`) are added unconditionally at top level (gated via `enabled`), interleaved with plain render-time computations — order stays stable every render. `hotGradesQ` is `enabled: featuredEnabled && hotCands.length>0`.

**Why:** the previous Featured Players row (and the dead `FeaturedPlayer`/`featured`/`featuredLoading` code, now removed) was a flat list; the redesign surfaces actionable, fully-honest betting value. Every rail self-hides when its real source is empty — never a placeholder.
